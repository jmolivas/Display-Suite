Display Suite

¿Que puede hacer Display Suite por mí ?

Display Suite permite controlar como los contenidos o campos de las entidades son desplegados utilizando una interfaz drag & drop. Sin la necesidad de crear o modificar archivos tpl.

Por defecto Display suite incluye vistas o modos predefinidos para mostrar los datos (Teaser, Full, RSS). Agregando nuevos View Modes es la manera de definir como las diferentes piezas de contenido o campos son mostradas en Teaser, Full ó Views

http://drupal.org/project/ds

Características

  • Agregar diferentes formatos por entidad (node, user, taxonomy)
  • Agregar nuevos campos utilizando tokens o codigo PHP.
  • Agregar bloques.
  • Opción de modificar etiquetas.
  • Agregar o reemplazar la configuración de estilos de campo (campos semánticos)
  • Integración con vistas y paneles.
  • Integración con WebForm.
  • Activando el Modulo Extras : Ocultar titulo, aplicar formato "View Mode" directamente a nivel de contenido.

  • Requiere CTools.

Context

¿Que puede hacer Context por mí ?

Context permite manejar condiciones contextuales y definir las reacciones que tendrán diferentes partes del sitio. Pensemos en cada contexto como representación de una "sección o página" dentro del sitio.

Para cada contexto, se definen las condiciones contextuales basadas en URL, Tipo de Contenido, Taxonomía, Lenguaje entre otros que lo invocan, además de definir las diferentes reacciones por ejemplo (asignar contenido en bloques) en el contexto activo.

http://drupal.org/project/context
